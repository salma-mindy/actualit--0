Snyk

une plate-forme de sécurité open-source

Ce rapport porte principalement sur les vulnérabilités et les risques en matière de sécurité dans les écosystèmes React et Angular.
Il parle en outre des :

  • Vulnérabilités de sécurité dans l’écosystème des frameworks Angular et React
  • Risques de sécurité liés aux dépendances indirectes
  • Sécurité Vue.js
  • Sécurité Bootstrap

En plus d’énumérer les vulnérabilités de sécurité dans React et Angular, le rapport partage également la posture de sécurité globale des deux. Cela comprend les conventions de codage sécurisé, les capacités sécurisées intégrées, les politiques de divulgation responsable et la documentation de sécurité dédiée au projet.

Vulnérabilités de sécurité dans l’écosystème des frameworks Angular et React

React

Dans le cas de React, Snyk a trouvé quatre packages malveillants, à savoir :
react-datepicker-plus, react-dates-sc , awesome_react_utility et reactserver-native.
Ceux-ci contiennent des codes malveillants qui récoltent des cartes de crédit et d’autres informations sensibles et attaquent des modules compromis sur l’écosystème React.
Bibliothèques vulnérables notables trouvés dans l’écosystème de React au cours de cette étude :
  • Le bibliothèque react-marked-markdown présente une vulnérabilité XSS de grande gravité qui n’a pas encore de correctif.
  • La bibliothèque preact-render-to-string est vulnérable à XSS dans toutes les versions antérieures à la version 3.7.2.
  • La bibliothèque react-svg a une vulnérabilité XSS de grande gravité qui a été divulguée par le chercheur en sécurité Ron Perris affectant toutes les versions antérieures à la version 2.2.18.
  • La bibliothèque mui-datatables présente la vulnérabilité CSV Injection.

Angular

Dans le cas de Angular, l’équipe de recherche Snyk a divisé l’écosystème Angular en trois domaines :
  • Modules d’écosystème Angular,
  • versions malveillantes de modules,
  • outils de développement.
Parlant des versions malveillantes des modules , le rapport énumère trois versions malveillantes pour les modules : angular-bmap, ng-ui-library et ngx-pica.
- La version « angular-bmap » 0.0.9 comprenait un code malveillant qui recueillait des renseignements de nature délicate sur les mots de passe et les cartes de crédit des formulaires. Il a ensuite utilisé pour envoyer ces informations à une URL contrôlée par un attaquant. Heureusement, cette version est maintenant retirée du registre npm.
- La version 1.0.987 de « ng-ui-library » a le même code malveillant que « angular-bmap » 0.0.9, bien qu’elle soit toujours maintenue.
- Le troisième module, « ngx-pica » (des versions 1.1.4 à 1.1.6), a également le même code malveillant que les deux modules ci-dessus.

Dans l’outillage des développeurs, le module angular-http-server était vulnérable à la traversée du répertoire deux fois.

L’écosystème du module angulaire comporte les modules vulnérables suivants :
ngx-bootstrap, ag-grid-community, ag-grid, ng-dialog, angular-gettext, angular-jwt, textangular, angular-froala, angular-redactor, i18n-node-angular.

Risques de sécurité liés aux dépendances indirectes

Les projets React et Angular sont souvent générés avec un outil d’échafaudage qui fournit une plaque chauffante. Dans React, nous utilisons la commande npm « create-react-app », alors que dans Angular, nous utilisons la commande npm « @angular/cli ». Dans un exemple de projet React et Angular créé à l’aide de ces outils d’échafaudage, il a été constaté que les deux incluaient des dépendances de développement avec des vulnérabilités. La bonne chose est qu’aucun d’eux n’avait de problèmes de sécurité de dépendance à la production.

« Il est important de noter qu’Angular s’appuie sur 952 dépendances, qui contiennent au total deux vulnérabilités; React s’appuie sur 1257 dépendances, contenant trois vulnérabilités et un problème potentiel de compatibilité de licence », indique le rapport.

Sécurité Vue.js

Au total, quatre vulnérabilités ont été détectées dans le projet de base Vue.js couvrant la période de décembre 2017 à août 2018 : trois expressions régulières moyennes et une faible vulnérabilité de déni de service.
Quant à l’écosystème du module de Vue, le rapport énumère les modules vulnérables suivants :
  • La bibliothèque bootstrap-vue présente une vulnérabilité XSS de grande gravité qui a été divulguée en janvier 2019 et a affecte toutes les versions antérieures
  • Une version malveillante a été publiée dans la bibliothèque vue-backbone.

Sécurité Bootstrap

L’équipe de recherche de Snyk a pu suivre un total de sept vulnérabilités XSS dans Bootstrap. Sur ces sept, trois ont été divulgués en 2019 pour les versions récentes de Bootstrap v3 et trois vulnérabilités de sécurité ont été divulguées en 2018, dont l’une affecte la nouvelle version 4.x Bootstrap. Toutes ces vulnérabilités ont des correctifs de sécurité et un chemin de mise à niveau pour les utilisateurs afin de remédier aux risques.
Parmi les modules vulnérables de l’écosystème Bootstrap figurent :
  • La bibliothèque bootstrap-markdown qui comprend une vulnérabilité XSS non indexée affectant toutes les versions.
  • La bibliothèque bootstrap-vuejs présente une vulnérabilité XSS de grande gravité qui affecte toutes les versions antérieures à bootstrap-vue 2.0.0-rc.12.
  • La bibliothèque bootstrap-select comprend également une vulnérabilité XSS de grande gravité.
